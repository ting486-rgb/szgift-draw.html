# 🌐 部署指南

## 方案1：Vercel（推荐）

1. 访问 [vercel.com](https://vercel.com)
2. 使用GitHub账号登录
3. 点击"New Project"
4. 上传项目文件夹或连接GitHub仓库
5. 部署完成后获得网址

## 方案2：Netlify

1. 访问 [netlify.com](https://netlify.com)
2. 注册并登录
3. 拖拽项目文件夹到部署区域
4. 自动部署并获得网址

## 方案3：GitHub Pages

1. 创建GitHub仓库
2. 上传所有项目文件
3. 在仓库设置中启用GitHub Pages
4. 选择main分支作为源
5. 获得 `username.github.io/repository-name` 网址

## 方案4：EdgeOne Pages

1. 在对话框上方点击"Integration"
2. 选择"EdgeOne Pages"并授权
3. 系统将自动部署项目

## 📁 需要部署的文件

确保以下文件都在项目根目录：

- `index.html` - 欢迎页面
- `personal-draw.html` - 主要抽签页面
- `admin-results.html` - 管理员页面（可选）
- `README.md` - 项目说明
- `package.json` - 项目配置

## 🔗 分享链接

部署完成后，您将获得一个网址，可以分享给团队成员：

```
https://your-domain.com
```

团队成员访问这个网址即可参与抽签！

## ⚠️ 注意事项

1. 数据存储在用户浏览器本地，不同设备间不共享
2. 建议在同一设备上完成抽签
3. 管理员功能需要使用 `admin` 账户登录
4. 如需重置，请联系管理员操作
